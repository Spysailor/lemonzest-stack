# Content Writer FR

Rédaction de contenu web SEO-friendly pour le marché francophone.

## Quand activer

- "Rédige-moi un article sur [sujet]"
- "J'ai besoin de contenu pour mon blog"
- "Page pilier sur [thématique]"
- "Fiche produit optimisée"
- "Landing page de présentation"
- "Rédaction web française"

## Structure type d'un article SEO FR

### 1. Brief préalable (à toujours demander si manquant)
Avant de rédiger, clarifier :
- **Mot-clé principal** ciblé
- **Intention de recherche** (informationnelle / transactionnelle / navigationnelle / commerciale)
- **Audience cible** (persona précis)
- **Longueur attendue** (500 / 1000 / 2000+ mots)
- **Ton** (professionnel / pédagogique / expert / décontracté)
- **CTA final** (lead magnet, contact, achat, partage)
- **Sources autorisées** ou références à citer

### 2. Structure standard
```
[Title optimisé — 50-60 caractères avec mot-clé]

[Meta description — 140-155 caractères, promesse + CTA]

[H1 — inclut le mot-clé principal, différent du title]

Introduction (80-150 mots)
- Hook (accroche émotionnelle ou chiffre marquant)
- Contexte du problème
- Promesse : ce que le lecteur va apprendre
- Teaser de la structure

[H2] Partie 1 — Sous-thème 1
   [H3] Point précis
   [H3] Point précis
   
[H2] Partie 2 — Sous-thème 2
   [H3] Point précis
   [H3] Point précis

[H2] Partie 3 — Sous-thème 3
   ...

[H2] Conclusion
- Synthèse des points clés
- Ouverture (étape suivante)
- CTA contextualisé
```

### 3. Densité mot-clé et sémantique
- **Mot-clé principal** : 1-2% de densité (pas de keyword stuffing)
- **Mots-clés secondaires** (LSI) : présents naturellement
- **Champ lexical** : couvrir 10-20 termes du sujet
- **Requêtes longue traîne** : inclure 3-5 questions utilisateurs

## Règles typographiques FR (NON-NÉGOCIABLES)

### Espaces insécables
Avant chaque signe de ponctuation double et devant les symboles :
- `Bonjour : voici un exemple` (espace insécable avant :)
- `15 %` (espace insécable avant %)
- `1 250 €` (espace insécable avant € et séparateur de milliers)
- `Chapitre 3` (espace insécable entre mot et chiffre qui le qualifie)

### Guillemets français
`« Comme ceci »` avec espaces insécables à l'intérieur.
Pas de `"guillemets droits"`.

### Apostrophe typographique
`L’agence` (pas `L'agence` avec apostrophe droite).

### Tirets
- Tiret cadratin `—` pour les incises : `Le SEO — c'est technique — demande de la rigueur.`
- Tiret demi-cadratin `–` pour les plages : `2024–2026`
- Trait d'union `-` pour les mots composés : `sous-skill`

### Points de suspension
`…` (caractère unique), pas `...` (3 points).

### Chiffres
- `1 250,50` (virgule décimale, espace insécable séparateur milliers)
- `1 000 000` (pas `1,000,000`)
- Nombres < 10 en lettres : `trois clients`, pas `3 clients`
- Exception : dates, heures, unités — chiffres toujours

### Dates et heures
- `17 avril 2026` (pas `17/04/2026` sauf tableaux)
- `14 h 30` (pas `14:30`, espace insécable)

## Ton selon le type de contenu

### B2B professionnel
- Vouvoiement par défaut
- Ton posé, argumenté, factuel
- Exemples concrets du secteur cible
- Chiffres et sources à l'appui

### B2C grand public
- Tutoiement possible selon la marque
- Ton chaleureux, accessible
- Storytelling accepté
- Images mentales, bénéfices émotionnels

### Technique / expert
- Vouvoiement
- Jargon maîtrisé (expliqué à la première occurrence)
- Exemples code / schéma
- Sources scientifiques / documentation officielle

### Éditorial / opinion
- Voix de l'auteur affirmée
- Tutoiement ou vouvoiement selon ligne éditoriale
- Prises de position argumentées
- Style plus littéraire permis

## Optimisations spécifiques FR

### Maillage interne
- 3-5 liens internes par article (anchors descriptives)
- Lier vers articles piliers de la même thématique
- Pas de "cliquez ici" : "découvrez notre guide sur [sujet]"

### Images
- Nommer les fichiers en FR : `audit-seo-wordpress.jpg` (pas `IMG_1234.jpg`)
- Alt text descriptif et francophone
- Format WebP ou JPG compressé
- Légende optionnelle si apporte valeur

### Format de lecture
- Paragraphes courts (3-5 lignes max)
- Liste à puces pour énumérations
- Encadrés / citations pour aérer
- Sous-titres H3 tous les 200-300 mots

### FAQ en fin d'article
Intégrer 3-5 questions-réponses courtes pour le schema FAQPage et la longue traîne.

## Conformité RGPD dans le contenu

- Ne pas mentionner de données clients nominatives sans accord
- Si capture email dans l'article : mentionner la politique de confidentialité
- Pour tout tracking : mentionner la gestion des cookies
- Si test / avis produit : préciser si affilié ou sponsorisé (transparence légale)

## Workflow recommandé

1. **Brief** — clarifier les 7 points du brief préalable
2. **Recherche sémantique** — lister 10-20 termes du champ lexical
3. **Plan détaillé** — H2/H3 avec 1 ligne de contenu prévu
4. **Validation plan** — soumettre au client avant rédaction
5. **Rédaction** — suivre le plan strictement
6. **Relecture typo FR** — espaces insécables, guillemets, apostrophes
7. **Optimisation SEO** — title, meta, Hn, mots-clés, liens
8. **Ajout FAQ** — 3-5 Q/R pour schema
9. **Livraison** — document formaté + consignes de publication

## Limites à respecter

- **Pas de copie** : toujours rédiger from scratch, jamais paraphraser un article existant
- **Pas d'invention** : citer des sources réelles, vérifiables
- **Pas de promesses creuses** : "doublez votre trafic" → conditionnel + conditions
- **Originalité** : vérifier via outil anti-plagiat si rédaction pour client payant
